title: "Get started with Passivbot"
description: "Configure API keys, choose a starting config, and run your first Passivbot live trading session in under 10 minutes with a step-by-step guide."
---

Once Passivbot is installed, getting to a live session takes four steps: add your exchange API keys, pick a starting configuration, run the bot, and confirm it is working. This guide walks through each step using the default trailing-grid configuration that ships with the repository.

<Steps>
  <Step title="Add your API keys">
    Passivbot reads exchange credentials from `api-keys.json` in the repository root. Copy the provided example file to create your own:

    ```bash
    cp api-keys.json.example api-keys.json
    ```

    Open `api-keys.json` and fill in your credentials. Each entry uses a short account name as the key — you will pass this name to the `passivbot live` command. Here is an example showing the structure for a few exchanges:

    ```json
    {
        "bybit_01": {
            "exchange": "bybit",
            "key": "your_api_key",
            "secret": "your_api_secret"
        },
        "okx_01": {
            "exchange": "okx",
            "key": "your_api_key",
            "secret": "your_api_secret",
            "passphrase": "your_passphrase"
        },
        "hyperliquid_01": {
            "exchange": "hyperliquid",
            "wallet_address": "0x_your_wallet_address",
            "private_key": "0x_your_private_key",
            "is_vault": false
        }
    }
    ```

    Exchanges that require a passphrase (Bitget, OKX, Kucoin) include a `"passphrase"` field. Hyperliquid uses wallet-based auth instead of API key/secret.

    <Warning>
      Never commit `api-keys.json` to version control. The file is listed in `.gitignore` by default, but verify this before pushing any changes.
    </Warning>
  </Step>

  <Step title="Choose a starting configuration">
    Passivbot ships with example configurations in `configs/examples/`. The recommended starting point is:

    ```
    configs/examples/default_trailing_grid_long_npos10.json
    ```

    This file mirrors the hardcoded defaults in `src/config/schema.py` exactly, so it is the most predictable starting point for new setups. Copy it to create your own working config:

    ```bash
    cp configs/examples/default_trailing_grid_long_npos10.json configs/my_config.json
    ```

    You can also run the bot without a config file at all — passing only `-u account_name` uses those same hardcoded defaults directly.
  </Step>

  <Step title="Run the bot">
    Start the bot using the `passivbot live` command. You have two options:

    **Default settings** — pass only your account name:

    ```bash
    passivbot live -u bybit_01
    ```

    **Custom config** — pass the path to your configuration file:

    ```bash
    passivbot live configs/my_config.json
    ```

    You can adjust log verbosity with `--debug-level` (alias `--log-level`):

    - `0` — warnings only
    - `1` — info (default)
    - `2` — debug
    - `3` — trace

    ```bash
    passivbot live -u bybit_01 --debug-level 1
    ```
  </Step>

  <Step title="Verify it's running">
    After startup, Passivbot logs its activity to the terminal. It also writes a timestamped log file under `logs/` and maintains a stable alias at `logs/{account_name}.log` that always points to the current run.

    To follow the live log for an account named `bybit_01`:

    ```bash
    tail -f logs/bybit_01.log
    ```

    Look for lines confirming the exchange connection, loaded configuration, and initial order placement. If you see errors about missing `passivbot_rust`, make sure your virtual environment is active and run `maturin develop --release` to rebuild the extension.
  </Step>
</Steps>

<Tip>
  Start with a small `wallet_exposure_limit` in your config (for example `0.02` for 2% of wallet balance per symbol) while you validate that the bot is behaving as expected. You can increase it later once you are comfortable with the strategy.
</Tip>
